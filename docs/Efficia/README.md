# Documentation Efficia™ — point d'entrée

**Source de vérité** de la Méthode Efficia™, consommée par le back-office, les workers Cloudflare, la base de données, les prompts IA et la génération des audits PDF.

## Comment naviguer

| Dossier | Rôle | On y trouve |
|---|---|---|
| `01 - Vision` | Le pourquoi de l'entreprise | Vision, mission, principes, positionnement |
| `02 - Méthode` | Le cadre intellectuel et documentaire | Méthode, 5 piliers, architecture, roadmap, changelog |
| `03 - Référentiel` | **LA source de vérité des faits** | Indicateurs, contrôles, observations, recommandations, secteurs |
| `04 - IA` | Le contrat homme-machine | Pipeline, prompts, variables, génération |
| `05 - Archives` | Le gelé | Versions passées de la méthode |

## Les deux règles d'or

1. **Une information n'existe qu'à un seul endroit.** Partout ailleurs, on la cite par son identifiant (`IND-012`, `CTL-042`…).
2. **Tout livrable est dérivé du référentiel.** Si un audit est faux, on corrige le référentiel, jamais le livrable.

Conventions complètes : voir `02 - Méthode/Architecture documentaire.md`.
